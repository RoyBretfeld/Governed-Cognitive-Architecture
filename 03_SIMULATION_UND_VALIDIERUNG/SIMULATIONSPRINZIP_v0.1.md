# Simulationsprinzip v0.1

Die Simulation läuft virtuell mit bekannten gelösten Fällen.

---

## Ziel

Prüfen, ob das Framework aus einer Ausgangslage selbst wieder zu einer tragfähigen Lösung kommt.

---

## Ablauf

```text
1. Gelösten echten Fall auswählen
2. Ausgangsdaten wiederherstellen
3. Bekannte Lösung ausblenden
4. Framework virtuell laufen lassen
5. Lösungsknoten erzeugen lassen
6. Ergebnis mit echter Lösung vergleichen
7. Lücken dokumentieren
```

---

## Bewertungsfragen

```text
- Wurde das Problem korrekt verstanden?
- Blieb die Bedeutung erhalten?
- Wurden mehrere Lösungswege geprüft?
- Wurden Nicht-Lösungen sinnvoll gespeichert?
- Wurde eine plausible Lösung gefunden?
- Wurde eine Checkliste erzeugt?
- Wurde korrekt eskaliert oder gelöst?
- Welche Daten hätten gefehlt?
```
